Keeping the metadata in sight
=============================

During the writing process, it can be useful to keep metadata such as chapter
contents, section summaries, character data or location descriptions in sight.
With *novelibre* you have various options for this.


The properties window
---------------------

The first choice is the `properties window <properties.html>`__.
You can drop down the properties you want to edit or view by clicking on the category.
Select the element to which the properties relate in the project tree.
The properties window can also be
`detached <view_menu.html#detach-dock-properties>`__
and placed separatey, e.g. at the edge of the screen.


Lists in the Web browser
------------------------

You can display separate lists for characters, locations, items and plot lines
in the web browser. In contrast to the properties window, you can see all
elements at once, but you cannot edit them.


Description documents
---------------------

For parts, chapters, sections, characters, locations, items, narrative structure and
plot lines, *novelibre* can export text documents that can be edited and reimported.
Usually, chapter headings are designed as links, which will take you directly to the
relevant place in the manuscript.


Tables
------

Tables are the classic tool from the time when specialized writing software
was not yet widespread.
The tables exported by *novelibre* are opened automatically with the *Calc*
spreadsheet software. They are---except the
`section list <section_menu.html#section-list-export-only>`__---editable
to be reimported by *novelibre*.
The section rows are linked to the sections in the manuscript.
Particularly recommended is the
`plot grid <plot_menu.html#export-plot-grid-for-editing>`__.

The only disadvantage of the tables is that you cannot add new tree elements
to *novelibre* with the spreadsheet.
